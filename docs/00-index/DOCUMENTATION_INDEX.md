## How to use this index

- Each document starts as a stub with frontmatter (title, category, status, priority, owner, last_updated).
- Update the `status` field as documents progress: Not Started -> Draft -> In Review -> Approved -> Living.
- High-priority documents should be completed before their corresponding implementation phase begins.
- This index itself should be updated whenever a document's status changes or a new document is added.
